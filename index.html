Ray Design System.

Introduction.

As Lightful continues to grow, both as a product and a company, one challenge we are faced with is learning how to refine the Lightful brand identity and apply it cohesively to each of our products. We created this styleguide to act as a central location where we house a live inventory of UI components, brand guidelines, brand assets and more. Anyone working on the Marvel product is encouraged to stay familiar with this styleguide and help ensure that it is kept up-to-date.

Cohesive Design Language

Refine our visual language to be simpler, more fun and more communicative.

Platform Agnostic System

Develop a single, unified system that is transferrable across all platforms and devices.

Expedite The Design Process

By abstracting our base styles layer, now developers can churn out UIs with minimal design input.

Typography.

Typefaces.

Lightful’s primary typeface is Euclid Flex — a simple typeface with legible numbers that renders well at all sizes. It can seem quite “trans­par­ent” when used in body text but dis­play some orig­i­nal traits when used in larger sizes. It’s famil­iar shape convays har­mony and ele­gance.

Uniform Spacing.

Spacing can be applied using either margin or padding. Both margin and padding share the same predefined scale. There are 4 available spacing sizes.

Grid

The grid is based on the Bootstrap Grid system which is a 12 column grid system. Components can vary from 2-8 columns.

Components.

Buttons.

Button Sizes

Buttons can have different sizes.

Button.

Primary buttons should be used to signify necessary or mandatory actions.

Outline Buttons.

Outline buttons should be used to signify optional, infrequent or subtle actions, like “cancel” or “go back”. Often paird with a flat button as a second option.

Disabled Button.

Disabled buttons don’t have hover states, and don’t change the mouse pointer on hover.
